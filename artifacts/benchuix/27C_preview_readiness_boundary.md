# BENCHUIX-27C Preview Readiness Boundary

## what would still be needed for browser visualization

- A separate BENCHUIX phase that explicitly authorizes local-only preview inspection.
- A preview procedure that remains candidate-only, synthetic-only and non-product.
- A review checklist that records no localhost drift, no package-manager drift and no real-state ambiguity during the visualization step.

## what 27C does not do

BENCHUIX-27C does not execute preview, browser or localhost. It creates only the synthetic test matrix, comprehension protocol, coverage audit, severity rubric, observation schema and fixture review.

## why preview requires a separate phase

Even a local-only preview changes the review surface from static source inspection to rendered behavior inspection. That requires its own gate, its own no-real-execution attestation and an explicit statement that the result is still not CRISOL or product admission.

## commands still prohibited in this phase

- `npm`
- `pnpm`
- `yarn`
- browser launch commands
- localhost servers
- preview commands
- package installation commands

## locks that remained false

- runtime execution
- preview execution
- browser execution
- localhost opened
- package manager execution
- real demo execution
- real user testing
- real apply
- product execution
- Bedrock execution
- secrets access
- real customer data
- real billing
- real OAuth
- real integrations
- live route opened
- real locks opened
- CRISOL opened

## future visualization rule

Any future visualization must remain local, synthetic, candidate-only and explicitly non-product. It cannot imply CRISOL admission, public deploy, real integration or any touched real state.
