# ARIS_CONTEXT_SYSTEM_LESSONS

## Problem

- Long conversations create confusion.
- ChatGPT memory is not source-of-truth.
- Obsidian drifted stale.
- `obsidian_notes_used = none` surfaced the gap.
- Very large prompts increase risk of drift and overclaiming.
- Token savings were not proven by evidence.
- F21-A showed that Context Intelligence needs a safer, derived navigation layer.

## Correction

- `aris-active-context` was created.
- 5 compact active files were established.
- History moved to `archive/`.
- Obsidian must become a derived navigation layer after repair.
- Context Law / Context Control must be fixed before F29.
- ChatGPT should consult active-context before ARIS decisions.
- The rejected external MCP path (`cyanheads/obsidian-mcp-server`) exposed write/delete/broad-vault risks.
- The preferred direction is an ARIS-owned read-only wrapper with query-first access and allowlisted paths.
- Historical next steps: wrapper spec, query-first contract, bulk-read prevention gate, ledger schema, shadow dry-run, readiness gate.

## Rules

- active-context before decision.
- `CURRENT_STATE.md` beats memory.
- `NEXT_ACTION.md` defines one action.
- `DECISION_LOCKS.md` blocks drift.
- Archive is not the default read.
- Obsidian is not primary source-of-truth.
- Token-saving claims require evidence.
- Every Codex prompt must update active-context.
- Obsidian sync becomes mandatory after the Context Law / Context Control repair.

- F28 repair: Obsidian Context Law / Context Control.
